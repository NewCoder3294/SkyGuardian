# iOS App — Build, TestFlight, and Device Test

Native SwiftUI app. Project generated by XcodeGen; built and shipped via the App
Store Connect API key (no Expo, no EAS).

- Bundle id: `com.nicolasdossantos.skyguardian`
- Team: `9KHR566436`
- ASC app: SkyGuardian (sku `skyguardian-ios`)

## Build & test
```bash
cd mobile
xcodegen generate
xcodebuild test -project ReconCompanion.xcodeproj -scheme ReconCompanion \
  -destination 'platform=iOS Simulator,name=iPhone 17'
```
Debug launch args (simulator): `-demo` (sample scene), `-feed` (point FEED at a
local backend on `:8011`).

## Ship to TestFlight (CLI)
ASC API auth lives in env (key in `~/.appstoreconnect/private_keys/`):
```bash
export ASC_ISSUER_ID=<issuer-uuid> ASC_KEY_ID=285VG6AQA5 \
       ASC_KEY_PATH="$HOME/.appstoreconnect/private_keys/AuthKey_285VG6AQA5.p8"

cd mobile
# bump CURRENT_PROJECT_VERSION in project.yml, then:
xcodegen generate
KEY="$ASC_KEY_PATH"
xcodebuild archive -project ReconCompanion.xcodeproj -scheme ReconCompanion \
  -destination 'generic/platform=iOS' -archivePath build/SkyGuardian.xcarchive \
  -allowProvisioningUpdates -authenticationKeyPath "$KEY" \
  -authenticationKeyID "$ASC_KEY_ID" -authenticationKeyIssuerID "$ASC_ISSUER_ID" \
  CODE_SIGN_STYLE=Automatic
xcodebuild -exportArchive -archivePath build/SkyGuardian.xcarchive -exportPath build/export \
  -exportOptionsPlist ExportOptions.plist -allowProvisioningUpdates \
  -authenticationKeyPath "$KEY" -authenticationKeyID "$ASC_KEY_ID" \
  -authenticationKeyIssuerID "$ASC_ISSUER_ID"
xcrun altool --upload-app -t ios -f build/export/ReconCompanion.ipa \
  --apiKey "$ASC_KEY_ID" --apiIssuer "$ASC_ISSUER_ID"
```
`scripts/asc.py` provisions bundle ids / lists apps (app *record* creation is web-UI
only — ASC API doesn't allow it). Processing takes ~5–15 min before the build
appears in TestFlight.

## Device test with the real Tello (single-network)
Put **both** the Mac and the iPhone on the **Tello's WiFi AP** (`192.168.10.x`).

**Mac:**
```bash
ipconfig getifaddr en0          # the Mac's IP on the Tello network
cd backend && source .venv/bin/activate
USE_MOCK=0 TELLO_SOURCE=tello uvicorn app.server:app --host 0.0.0.0 --port 8011
# wait for: [video] tello source connected
```
**iPhone (TestFlight → SkyGuardian):**
1. MISSION LINK → set `ws://<mac-ip>:8011/ws` (the IP from `ipconfig`, **not** 127.0.0.1).
2. CONNECT → map/status go live. Tap **FEED** → live Tello camera.

Gotchas: the app default URL is `127.0.0.1` (simulator) — change it on the phone;
keep the phone on the Tello AP even with no internet; voice shows UNAVAILABLE until
the Cactus framework + model are added (see VOICE.md).
